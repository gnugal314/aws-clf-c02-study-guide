# Daily Practice Questions

## Day 1 — Cloud Fundamentals

1. What cloud benefit eliminates upfront server purchases?
   - Answer: Pay-as-you-go pricing.
2. What is elasticity?
   - Answer: Automatically scaling resources up and down based on demand.
3. What is scalability?
   - Answer: The ability to handle growth in workload.
4. What is OpEx?
   - Answer: Operational expense based on ongoing usage.
5. What is CapEx?
   - Answer: Upfront capital expense for assets like hardware.
6. What AWS benefit improves deployment speed?
   - Answer: Agility.
7. Why do organizations move to cloud computing?
   - Answer: Flexibility, scalability, speed, and reduced infrastructure management.
8. What does on-demand mean?
   - Answer: Resources are available when needed.
9. Which pricing model avoids long-term commitment?
   - Answer: On-Demand pricing.
10. What business problem does elasticity solve?
   - Answer: Overprovisioning and underprovisioning.

## Day 2 — Infrastructure

1. What is an AWS Region?
   - Answer: A geographic AWS location with multiple Availability Zones.
2. What is an Availability Zone?
   - Answer: One or more isolated data centers within a Region.
3. Why use multiple AZs?
   - Answer: To improve availability and fault tolerance.
4. What is hybrid cloud?
   - Answer: A combination of cloud and on-premises resources.
5. What is high availability?
   - Answer: Designing systems to minimize downtime.
6. What is fault tolerance?
   - Answer: Continuing to operate despite failures.
7. What AWS service helps deliver content globally?
   - Answer: Amazon CloudFront.
8. What is data residency?
   - Answer: Keeping data within required geographic boundaries.
9. What is a private cloud?
   - Answer: Dedicated infrastructure for one organization.
10. What improves global disaster recovery?
   - Answer: Multi-Region architecture.

## Day 4 — Accessing AWS

1. What is the AWS Management Console?
   - Answer: A web-based interface to manage AWS services.
2. What is AWS CLI?
   - Answer: A command-line tool for AWS service management.
3. What is an SDK?
   - Answer: A language-specific library for using AWS in applications.
4. Which tool is best for beginners?
   - Answer: AWS Management Console.
5. Which tool supports automation scripts?
   - Answer: AWS CLI.
6. Which tool is browser-based and command-line oriented?
   - Answer: AWS CloudShell.
7. Which tool is best for application integration?
   - Answer: AWS SDK.
8. What is programmatic access?
   - Answer: Accessing AWS using APIs, CLI, or SDKs.
9. What is the Console Mobile Application used for?
   - Answer: Viewing and managing AWS resources from mobile devices.
10. Why use CLI instead of Console?
   - Answer: Automation and repeatability.

## Day 7 — Databases

1. What is RDS?
   - Answer: Managed relational database service.
2. What is DynamoDB?
   - Answer: Serverless NoSQL key-value database.
3. What is Aurora?
   - Answer: AWS high-performance relational database.
4. What is Redshift?
   - Answer: Cloud data warehouse.
5. What is ElastiCache?

6. What is Neptune?

7. What is DocumentDB?

8. Which database is best for SQL applications?

## Day 8 — Networking

1. What is VPC?
   - Answer: A logically isolated private AWS network.
2. What is Route 53?
   - Answer: AWS DNS service.
3. What is CloudFront?
   - Answer: AWS content delivery network.
4. What is ELB?
   - Answer: A service that distributes traffic across targets.
5. What is Direct Connect?
   - Answer: A dedicated private connection to AWS.
6. What is Site-to-Site VPN?
   - Answer: Encrypted network tunnel to AWS.
7. What is Transit Gateway?
   - Answer: A central hub for connecting VPCs and networks.
8. What is a subnet?
   - Answer: A segment of a VPC.
9. What is an Internet Gateway?
   - Answer: A component that allows internet access for a VPC.
10. What is a NAT Gateway?
   - Answer: Allows private subnet resources outbound internet access.
